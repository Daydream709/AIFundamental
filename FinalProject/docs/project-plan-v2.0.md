# 数据科学与工程课程 Project Two 最终执行方案

**项目名称**：多变量长周期时间序列预测系统：架构对比、自研高性能模型与轻量化探索


## 一、研究问题与核心目标

本项目旨在回答四个层面的核心问题：

1. **架构层面**：MLP、Transformer、CNN、SSM 四大架构在不同维度（低/中/高）和不同领域（电力/气象）下的本质优劣势是什么？
2. **性能极限层面**：通过集成 KAN 网络、频域分解、概率预测等前沿优化手段，自研高性能模型能否达到该场景下的精度上限？
3. **效率权衡层面**：如何在保持极低参数量（<0.05M）的前提下，通过结构创新逼近 SOTA 精度？
4. **多模态有效性层面**：在 Environment 数据集上，文本模态（环境报告 vs 搜索摘要）能否为空气质量预测带来显著增益？该增益是普遍现象还是仅对特定架构有效？

## 四、两大自研模型详细设计

### 4.1 自研高性能模型：KAN-iTransformer（冲刺精度上限）

在 iTransformer 的倒置架构基础上，集成以下 **5+2=7 大优化模块**（★ 为 v2.1 新增）：

- **模块1（架构增强 · 真实 B-spline KAN）** ：将原 FFN 替换为 **KAN 层**，**v2.1 升级为完整 Cox-de Boor 递推 B-spline**（替代 v1 的"伪KAN"），基函数带 spline_weight 可学习控制点权重。引入 **级联频域分解（CFD）** ，逐层剥离趋势/季节/残差，交由不同 KAN 专家处理。
- **模块1+（★ Wavelet-CFD 替代 FFT-CFD）** ：v2.1 新增 **Wavelet 版频域分解** (PyWavelets, db4 小波, level=2)。FFT 假设全局平稳，Wavelet 在时频两域都有分辨率，能更好捕捉非平稳信号（如电力突变）。通过 `--use_wavelet` 开关切换。
- **模块2（预训练策略）** ：正式训练前增加 **掩码重建自监督任务**（掩码率15%），让模型学习数据内在表征，提升鲁棒性。
- **模块3（概率输出）** ：训练损失改用 **GaussianNLL**（输出均值+方差），使模型具备不确定性建模能力；推理后增加 **共形预测（Conformal Prediction）** 校准，输出理论保证的95%置信区间。
- **模块4（归一化）** ：引入 **RevIN（可逆实例归一化）** ，消除训练-测试间的分布偏移。
- **模块5（模型仲裁）** ：提取输入序列5维统计特征（谱熵/趋势强度/周期性/方差/自相关），训练轻量 MLP 路由器，动态融合 KAN-iTransformer、PatchTST、Mamba 三者的预测结果。

> **v2.1 升级亮点**：
> 1. KAN 从"伪KAN"（spline_weight 取平均作为 Linear 近似）升级为**完整 B-spline**，非线性拟合能力真实跃升
> 2. 新增 Wavelet 版频域分解，适合突变/非平稳信号
> 3. 新增开关 `--use_wavelet` 启用 Wavelet-CFD（默认仍为 FFT-CFD，可复现旧结果）

### 4.2 自研轻量化模型：Lite-SparseNet（冲刺效率极限）

针对 SparseTSF 忽略多变量关联的缺陷，提出以下 **3+2=5 大创新**（★ 为 v2.1 新增）：

- **阶段一（稀疏趋势提取 · 共享权重 + 变量偏置）** ：借鉴 SparseTSF，对每个变量独立进行跨周期下采样（长度 H 压缩为 H/p），捕获宏观趋势，大幅降低序列长度。**v2.1 升级**：所有变量共享同一 Linear 权重，差异通过 variable-specific bias 表达 —— **参数量从  enc_in×down_len×pred_len 降到 down_len×pred_len+enc_in**。
- **阶段二（轻量变量间交互）** ：下采样后，引入 **分组轻量 MLP（Group-wise Light MLP）** 。将高维变量分成若干组（如每组16个），仅在组内进行轻量化信息交互，捕获变量间协同变化，参数量增量仅为 `组数 × 16 × 16`，远低于全连接。
- **阶段三（频域残差修正）** ：在时域预测基础上，增加单层 FFT 残差模块，只捕捉最重要的1-2个主频分量来修正趋势预测的细节误差（计算复杂度 O(H log H)，几乎不引入额外参数）。
- **阶段0（★ Lite-RevIN 极简归一化）** ：v2.1 新增。在最前面做极简实例归一化 `(x - mean) / std`，推理时反归一化。**零可学习参数**，解决变量间尺度差异巨大的问题（电力与温度不能放同一量级）。
- **阶段0-（★ 共享偏置）** ：v2.1 新增。趋势提取后，对每个变量加 1 个可学习 bias 恢复变量特异性。参数量 = enc_in。

> **v2.1 升级亮点**：
> 1. **共享权重** + 偏置 —— 在 ETTm2 上参数量从 ~18K 降到 **~2.7K**（约 6-7 倍压缩），仍维持 < 0.05M 约束
> 2. **Lite-RevIN** 极简实例归一化，让模型更稳定，跨数据集泛化更好
> 3. 在 Electricity (321 维) 上，参数量 ~23K，**远低于 50K 上限**，仍有空间加入更多优化


## 五、实验主线设计

将实验划分为四条清晰的主线（三大主线 + 消融实验专项），确保逻辑严密、互不重叠：

- **主线一（全架构对比）**：广度对比 4 大架构在不同数据集的表现
- **主线二（自研模型评测）**：深度验证自研 KAN-iTransformer / Lite-SparseNet
- **主线三（多模态消融）**：聚焦多模态有效性，定量回答 4 个核心问题
- **主线四（消融实验专项）**：聚焦最关键的 3 类消融（KAN 5 模块 / Lite 3 阶段 / 多模态 5 组）

### 主线一：全架构对比实验（广度）

- **目的**：分析 MLP、Transformer、CNN、SSM 四大架构在不同数据特性下的表现。
- **模型**：DLinear, PatchTST, TimesNet, Mamba。
- **数据集**：ETTm2, Weather, Electricity。
- **预测长度**：{96, 192, 336, 720}。
- **输出**：架构优劣势分析报告（如 Transformer 擅长长程依赖但高维计算量大，CNN 擅长局部模式提取，Mamba 在效率与效果间取得平衡等）。

### 主线二：自研模型深度评测（深度）

- **目的**：验证自研高性能和轻量化模型的有效性。
- **模型**：KAN-iTransformer（高性能）、Lite-SparseNet（轻量化）、SparseTSF（外部轻量化标杆）、DLinear（底线参考）。
- **数据集**：全部4个数据集（ETTm2, Weather, Electricity, Environment）。
- **核心分析**：
  1. KAN-iTransformer 是否在所有设置下达到最佳 MSE/MAE？
  2. Lite-SparseNet 是否在参数量（50K）远小于大模型的同时，在 Environment（6维）上表现显著优于 SparseTSF（1K）并逼近 PatchTST（6.9M）？
  3. 对比 Weather（21维纯数值）和 Environment（6维+文本）上各模型的性能差异——同样是环境/气象领域，低维度加文本能否弥补纯数值信息的不足？

### 主线三：多模态有效性消融（聚焦）

- **场景**：仅在 **Environment** 数据集上，固定预测长度（如 96 和 192）。
- **文本模态说明**：Environment 自带两类文本——
  - `report`：环境报告（宏观政策/年度总结，约156条）
  - `search`：相关搜索摘要（公众关注度，约2,272条）
- **对比设置（5组）** ：

| 实验组 | 输入模态 | 说明 |
|--------|---------|------|
| 组1（基线） | 仅时序数值（6个变量） | 纯数值基准 |
| 组2 | 时序 + `report` 文本 | 验证宏观背景知识是否有助于预测 |
| 组3 | 时序 + `search` 文本 | 验证公众关注度/实时舆情是否更有用 |
| 组4 | 时序 + `report` + `search`（简单拼接） | 验证堆叠文本是否带来额外增益 |
| 组5 | 时序 + 融合文本（门控融合） | 验证自适应门控融合是否优于简单拼接 |

- **参与模型（选2个代表性架构）** ：PatchTST（Transformer 代表）、SparseTSF（外部轻量化标杆）。
- **核心分析问题**：
  1. 文本是否有效？→ 组1 vs 组2/3，量化 MSE/MAE 的降低幅度。
  2. 哪种文本更有用？→ 组2 vs 组3，分析 `report`（宏观）与 `search`（微观）的贡献差异（假设：`search` 实时舆情对短期预测更有帮助）。
  3. 融合方式重要吗？→ 组4 vs 组5，验证门控机制是否优于简单拼接。
  4. 增益是否模型无关？→ 比较三种模型上文本带来的增益是否一致。


### 主线四：消融实验（聚焦 · 三组最关键消融）

**设计原则**：消融实验的目标是**证明每个设计选择都"有道理"**，而非简单罗列所有超参。聚焦"论文审稿人最关心"的 3 类消融：

#### 消融 1：KAN-iTransformer 5 大模块的贡献度

- **目的**：验证 KAN-iTransformer 5 大模块（KAN+CFD / 掩码预训练 / 概率输出 / RevIN / 模型仲裁）**每一个都不可替代**。
- **数据集**：ETTm2 + Electricity（最具代表性：低维 vs 高维）
- **基线（A0）**：完整 KAN-iTransformer（5 模块全开）
- **消融组**（A1-A6，每次关掉 1 个模块）：

| 消融组 | 关掉的模块 | 替换为 | 预期影响 |
|--------|-----------|--------|---------|
| **A0 · 完整版** | 无 | — | 最佳 MSE (基准) |
| **A1** | KAN 层 | 传统 FFN (Linear→ReLU→Linear) | **最大** - 验证 KAN 非线性拟合 |
| **A2** | 级联频域分解 (CFD) | 单次三分支不逐层剥离 | 验证"逐层剥离"必要性 |
| **A3** | 掩码预训练 | 跳过 `pretrain_step` 直接训练 | 验证预训练对泛化帮助 |
| **A4** | 概率输出 (GaussianNLL) | 改用 MSE + 单点预测 | 验证不确定性建模价值 |
| **A5** | RevIN 归一化 | 简单实例归一化 (均值-标准差) | 验证分布偏移纠正效果 |
| **A6** | 模型仲裁 | 单模型直接预测 (取均值) | 验证多模型动态融合收益 |

- **核心分析问题**：
  1. 哪些模块是"必须有"的 (Δ MSE > 5%)？
  2. 哪些模块是"锦上添花"的 (Δ MSE < 2%)？
  3. 模块间的相互作用 — 去掉 A1 后 A4 仍有效吗？
- **输出**：5 模块消融表 (7×2 矩阵, 7 行 = A0-A6, 2 列 = MSE/MAE)。

#### 消融 2：Lite-SparseNet 3 阶段设计的贡献度

- **目的**：验证 Lite-SparseNet 三阶段设计（稀疏趋势 / 分组 MLP / FFT 残差）的**每一阶段都不可替代**，尤其在 < 0.05M 极低参数量约束下。
- **数据集**：Environment (6 维) + Electricity (321 维) — Lite 在小维度和大维度都需验证
- **基线（B0）**：完整 Lite-SparseNet（3 阶段全开, ~0.018M）
- **消融组**（B1-B3）：

| 消融组 | 关掉 | 替换为 | 验证重点 |
|--------|------|--------|---------|
| **B0 · 完整版** | 无 | — | 基准 (~0.018M) |
| **B1** | S2 分组 MLP | 每个变量独立线性 (无交互) | 验证变量间交互必要 |
| **B2** | S3 FFT 残差 | 去掉残差修正 | 验证频域细节捕获价值 |
| **B3** | 全部 (退化为稀疏版 Lite) | 近似 SparseTSF | **关键对比** — Lite-SparseNet vs SparseTSF 谁优 |

- **核心分析问题**：
  1. S2 (分组 MLP) 在 6 维 vs 321 维上的贡献差异 (变量越多，交互越重要)
  2. S3 (FFT 残差) 是否真的"零参数"却"有效果"
  3. B3 vs SparseTSF 直接对比 — 自研是否真比基线强
- **输出**：3 阶段消融表 + 与 SparseTSF 的关键对比图。

#### 消融 3：多模态 5 组文本融合（= 主线三，但作为消融的定量分析）

- **目的**：**严格量化** 5 组消融下文本模态对预测精度的具体贡献。
- **数据集**：Environment (6 维 + 文本)
- **参与模型**：PatchTST (Transformer 代表) + SparseTSF (轻量代表) — 各代表一类架构
- **5 组消融**（与主线三保持一致）：

| 组 | 输入模态 | 验证内容 | 期望影响方向 |
|----|---------|---------|-------------|
| **C1 (基线)** | 仅时序数值 | 纯数值 baseline | 基准 MSE |
| **C2** | + report (宏观) | 宏观报告是否有用 | report 数据量少 (156 条)，可能帮助有限 |
| **C3** | + search (实时舆情) | 公众关注度是否更有用 | 假设 + (search 数据多, 实时) |
| **C4** | + report + search 简单拼接 | 多源叠加 vs 单源 | 假设 + (但边际递减) |
| **C5** | + 门控融合 (gating) | 自适应 vs 拼接 | **关键** — 假设 gate > 简单拼接 |

- **核心分析问题（带量化指标）**：
  1. **文本是否有效？** → C1 vs C2/C3, 量化 Δ MSE (%)
  2. **哪种文本更有用？** → C2 vs C3, 量化 report vs search 的贡献差
  3. **融合方式重要吗？** → C4 vs C5, 量化门控 vs 拼接的差异
  4. **增益是否模型无关？** → PatchTST 和 SparseTSF 上分别计算 Δ MSE
- **输出**：5 组消融表 + 4 个核心问题的量化回答。

#### 与统计检验结合

消融实验的"完整版 vs 消融版"使用 **Wilcoxon 符号秩检验**（α=0.05）确认差异显著：
- 原假设 H0：消融版 MSE 与完整版无显著差异
- 接受 H0 (p > 0.05)：该模块贡献不显著
- 拒绝 H0 (p ≤ 0.05)：该模块贡献显著，论文可写"显著优于"


## 六、评估指标与效率统计

- **预测精度指标**：MSE, MAE, RMSE, MAPE, SMAPE。
- **效率指标**：参数量 (Params)、推理时间 (ms)、GPU显存峰值 (MB)，绘制 **帕累托前沿图**（精度 vs 效率）。
- **统计检验**：对主要结论（如 KAN-iTransformer vs PatchTST）进行 **Wilcoxon 符号秩检验**（α=0.05），确保性能提升非偶然。
