# 数据科学与工程课程 Project Two 修订版实验计划书（v2.0）

**项目名称**：多变量长周期时间序列预测系统：架构对比、自研高性能模型与轻量化探索


## 一、研究问题与核心目标（微调）

本项目旨在回答四个递进层面的核心问题：

1. **架构层面**：MLP、Transformer、CNN、SSM 四大架构在**不同维度（低/中/高）** 和**不同领域（电力/气象）** 下的本质优劣势是什么？
2. **性能极限层面**：通过集成 KAN、频域分解、概率预测等前沿优化，自研高性能模型能否达到该场景下的精度上限？
3. **效率权衡层面**：如何在保持极低参数量（<0.05M）的前提下，逼近 SOTA 精度？
4. **多模态有效性层面（新增重点）** ：在 Environment 数据集上，**文本模态（环境报告+搜索摘要）** 是否能为空气质量/天气预测带来显著增益？这种增益是普遍现象，还是仅对特定架构（如 Transformer）有效？

## 四、两大自研模型详细设计（保持不变）

### 4.1 自研高性能模型：KAN-iTransformer（冲刺精度上限）

集成4大优化模块：
- **模块1（架构增强）** ：KAN层替换FFN + 级联频域分解（CFD）逐层剥离趋势/季节/残差。
- **模块2（概率输出）** ：GaussianNLL 训练损失 + 共形预测（Conformal Prediction）输出95%置信区间。
- **模块3（归一化）** ：RevIN（可逆实例归一化）。
- **模块4（模型仲裁）** ：5维统计特征（谱熵/趋势强度/周期性/方差/自相关）+ 轻量MLP路由器，动态融合 KAN-iTransformer、PatchTST、Mamba 三者的预测。

### 4.2 自研轻量化模型：Lite-SparseNet（冲刺效率极限）

两点原创改进：
- **原创点1**：稀疏下采样后引入**分组轻量MLP（Group-wise Light MLP）** ，以极小参数量（组数×16×16）捕获变量间交互。
- **原创点2**：单层FFT残差模块，捕捉1-2个主频分量修正趋势预测细节误差。


## 五、实验主线设计（主线三重点修订）

### 主线一：全架构对比实验（广度）—— **保持不变**
- **目的**：分析 MLP、Transformer、CNN、SSM 四大架构在不同数据特性下的表现。
- **模型**：DLinear, PatchTST, TimesNet, Mamba。
- **数据集**：ETTm2, Weather, Electricity。
- **预测长度**：{96, 192, 336, 720}。
- **输出**：架构优劣势分析报告。

### 主线二：自研模型深度评测（深度）—— **数据集更新为4个**
- **目的**：验证自研高性能和轻量化模型的有效性。
- **模型**：KAN-iTransformer（高性能）、Lite-SparseNet（轻量化）、SparseTSF（外部轻量化标杆）、DLinear（底线参考）。
- **数据集**：全部4个数据集（**ETTm2, Weather, Electricity, Environment**）。
- **核心分析**：
  1. KAN-iTransformer 是否在所有设置下达到最佳 MSE/MAE？
  2. Lite-SparseNet 是否在参数量（50K）远小于大模型的同时，**在 Environment（6维）上表现优于 SparseTSF（1K）并逼近 PatchTST（6.9M）**？
  3. **新增分析**：对比 Weather（21维纯数值）和 Environment（6维+文本）上各模型的性能差异——同样是环境/气象领域，**维度低+文本**能否弥补纯数值信息的不足？

### 主线三：多模态有效性消融（聚焦修订）—— **重点修改**
- **场景**：仅在 **Environment** 数据集上，固定预测长度（如 96 和 192）。
- **文本模态说明**：Environment 自带两类文本——
  - `report`：环境报告（宏观政策/年度总结，约156条）
  - `search`：相关搜索摘要（公众关注度，约2,272条）
- **对比设置（5组，新增了文本类型细粒度分析）** ：

| 实验组      | 输入模态                               | 说明                                 |
| ----------- | -------------------------------------- | ------------------------------------ |
| 组1（基线） | 仅时序数值（6个变量）                  | 纯数值基准                           |
| 组2         | 时序 + `report` 文本                   | 验证“宏观背景知识”是否有助于预测     |
| 组3         | 时序 + `search` 文本                   | 验证“公众关注度/实时舆情”是否更有用  |
| 组4         | 时序 + `report` + `search`（简单拼接） | 验证“堆叠文本”是否带来额外增益       |
| 组5         | 时序 + 融合文本（门控融合）            | 验证“自适应门控融合”是否优于简单拼接 |

- **参与模型（选3个代表性架构）** ：
  - **PatchTST**（Transformer 代表）
  - **KAN-iTransformer**（自研高性能代表）
  - **Lite-SparseNet**（自研轻量化代表）
- **核心分析问题**：
  1. **文本是否有效？** → 组1 vs 组2/3，量化 MSE/MAE 的降低幅度。
  2. **哪种文本更有用？** → 组2 vs 组3，分析 `report`（宏观）与 `search`（微观）的贡献差异。*假设：search（实时舆情）可能比 report（年度总结）对短期预测更有帮助。*
  3. **融合方式重要吗？** → 组4 vs 组5，验证你设计的门控机制是否优于简单拼接。
  4. **增益是否模型无关？** → 比较三种模型上文本带来的增益是否一致，还是说大模型（KAN-iTransformer）更能利用文本信息？
