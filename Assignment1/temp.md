### 一、技术层面对比：核心能力测试Prompt与方法

#### 1. 基础语言能力测试（通用场景）

##### 测试目标

验证模型的语义理解、逻辑表达、无幻觉能力，适用于所有通用大模型对比。

##### 标准化Prompt（直接复制使用）

| 测试类型   | 测试Prompt                                                                                                                                                    | 评估维度（1-10分）                                               |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| 语义理解   | "请解释以下概念：向量数据库，要求说明其核心作用、典型应用场景（至少2个），并对比传统关系型数据库的差异（至少3点）。"                                          | 准确性（概念无错误）、完整性（覆盖所有要求）、逻辑性（对比清晰） |
| 逻辑推理   | "有A、B、C、D四个人，已知：1. A比B大3岁；2. C的年龄是D的2倍；3. B比C小5岁；4. 四人年龄总和为82岁；5. D的年龄是10岁。请计算每个人的年龄，并写出详细推导步骤。" | 步骤完整性、计算准确性、逻辑连贯性                               |
| 无幻觉测试 | "请介绍2025年发布的Python 4.0版本的核心新特性，要求列出至少3个官方确认的特性，并注明特性来源（如PEP编号、官方文档链接）。"                                    | 无幻觉率（若模型编造不存在的特性则扣分）、信息准确性             |
| 中文创作   | "以'数字时代的乡愁'为主题，写一篇300字左右的散文，要求语言流畅、有情感共鸣，避免空洞抒情。"                                                                   | 文采（语言优美度）、主题契合度、原创性                           |

##### 测试方法

- 对两个模型输入**完全相同的Prompt**，禁用上下文记忆（单轮测试）。
- 按评估维度打分（1-10分），计算各维度平均分，分数差≥2分则判定为显著差异。
- 记录模型是否出现幻觉（编造事实）、答非所问、内容重复等问题。

#### 2. 专业领域能力测试（代码/数学/垂直场景）

##### 代码生成测试

**Prompt**：

```

**评估标准**：

- 语法正确性（是否有语法错误）；
- 功能完整性（是否覆盖所有要求）；
- 异常处理（是否考虑文件不存在等情况）；
- 可读性（注释是否清晰、代码结构是否规范）。

##### 数学推理测试

**Prompt**：

```
求解以下数学问题，给出详细推导步骤：

### 二、服务响应层面对比：性能/成本/体验测试方法

#### 1. 性能指标测试（量化）

##### 测试工具

- 手动测试：使用Python的`time`模块记录请求耗时；
- 批量测试：使用`locust`进行压测（适合高并发场景）。

##### 测试代码（基础耗时统计）

```python
import openai
import time
import os

# 配置API信息
openai.api_key = os.getenv("OPENAI_API_KEY")
# 国内模型需配置API_BASE
# openai.api_base = "https://dashscope.aliyuncs.com/compatible-mode/v1"

def test_response_performance(model_name, prompt, test_times=10):
    """
    测试模型响应性能
    :param model_name: 模型名称（如gpt-3.5-turbo、qwen-turbo）
    :param prompt: 测试用Prompt
    :param test_times: 测试次数（取平均值）
    :return: 平均首字延迟、平均端到端时延
    """
    ttft_list = []  # 首字延迟（Time To First Token）
    total_time_list = []  # 端到端时延

    messages = [{"role": "user", "content": prompt}]

    for i in range(test_times):
        try:
            start_time = time.time()
            # 流式调用（用于统计首字延迟）
            response = openai.ChatCompletion.create(
                model=model_name,
                messages=messages,
                stream=True,  # 开启流式输出
                temperature=0.7
            )

            first_token_time = None
            full_response = ""
            for chunk in response:
                if chunk.choices[0].delta.content and first_token_time is None:
                    # 记录第一个Token的时间
                    first_token_time = time.time()
                    ttft = first_token_time - start_time
                    ttft_list.append(ttft)
                if chunk.choices[0].delta.content:
                    full_response += chunk.choices[0].delta.content

            # 记录端到端时延
            total_time = time.time() - start_time
            total_time_list.append(total_time)

        except Exception as e:
            print(f"第{i+1}次测试失败：{e}")
            continue

    # 计算平均值
    avg_ttft = sum(ttft_list) / len(ttft_list) if ttft_list else 0
    avg_total_time = sum(total_time_list) / len(total_time_list) if total_time_list else 0

    print(f"模型：{model_name}")
    print(f"平均首字延迟：{avg_ttft:.2f}秒")
    print(f"平均端到端时延：{avg_total_time:.2f}秒")
    print(f"单Token平均时延：{(avg_total_time / len(full_response)):.4f}秒/Token")

    return avg_ttft, avg_total_time

# 调用测试函数
if __name__ == "__main__":
    test_prompt = "请详细解释Python中的异步编程原理，包括协程、事件循环、await关键字的作用，字数不少于500字。"
    # 测试模型1
    test_response_performance("gpt-3.5-turbo", test_prompt)
    # 测试模型2
    test_response_performance("qwen-turbo", test_prompt)
```

##### 核心性能指标定义

- **首字延迟（TTFT）**：从发送请求到收到第一个Token的时间（越小越好，影响用户体感）；
- **端到端时延**：从发送请求到收到完整回复的总时间；
- **单Token时延（TPOT）**：端到端时延 ÷ 生成的Token数（衡量生成流畅度）。

| 流式输出体验   | 开启流式输出，观察回复是否实时显示、是否卡顿、是否乱序                                                       | 流畅度评分（1-10分）：实时性、无卡顿、无乱序           |

### 三、测试注意事项

1. **控制变量**：测试时确保两个模型的参数一致（temperature、max_tokens、top_p等），避免参数差异影响结果；
2. **多次测试取平均**：单次测试可能受网络波动影响，建议每个Prompt测试5-10次，取平均值；
3. **记录环境信息**：测试时记录网络环境（带宽、地区）、硬件配置，保证测试可复现；
4. **标准化评估**：使用统一的评分表（1-10分），避免主观判断偏差。

### 总结

1. 技术层面测试核心是**标准化Prompt+量化评分**，重点验证模型的准确性、完整性、专业性；
2. 服务响应层面测试需结合**代码统计（性能）+ 定价计算（成本）+ 体验评分（定性）**，全面对比；
3. 所有测试需遵循**控制变量法**，确保两个模型在相同条件下对比，结果才具备参考价值。
