Python 3.12 新特性展示

探索Python 3.12的三个重要更新
Python 3.12带来了多项性能改进和新特性，使代码更高效、更易读。以下展示三个最值得关注的新特性。
🐞

更友好的错误信息

增强的错误提示和回溯信息
Python 3.12改进了错误信息的可读性，特别是在涉及字典键、属性访问和导入错误时。错误信息现在会明确指出问题所在，并提供更清晰的建议。
示例代码：改进的错误信息
# 示例1: 字典键错误
my_dict = {"name": "Alice", "age": 30}

# 在Python 3.12中，错误信息会明确指出哪个键不存在
print(my_dict["nam"])  # 注意：应该是"name"而不是"nam"

# 示例2: 属性错误
class Person:
    def __init__(self, name):
        self.full_name = name

p = Person("Bob")
# 更清晰的错误信息会提示可能的正确属性名
print(p.name)  # 应该是full_name

# 示例3: 导入错误
# 尝试导入不存在的模块
import matplotlb  # 应该是matplotlib

PEP 701: f-string语法形式化

更灵活、更强大的f-string表达式
Python 3.12正式采纳PEP 701，解除了f-string表达式的许多限制。现在可以在f-string中使用反斜杠、多行表达式和相同引号嵌套。
示例代码：增强的f-string功能
# 示例1: f-string中使用反斜杠（之前不允许）
items = ["apple", "banana", "cherry"]
print(f"列表项: {', '.join(items)}")

# 示例2: 多行表达式和注释
x = 10
y = 20
result = f"""
总和: {x + y}
差值: {x - y}
乘积: {x * y}
商: {x / y if y != 0 else 'undefined'}
"""
print(result)

# 示例3: 相同引号嵌套（之前需要转义）
name = "Alice"
# 现在可以在f-string中使用相同类型的引号
print(f'他说: "你好, {name}!"')

# 示例4: 更复杂的表达式
def double(n):
    return n * 2

numbers = [1, 2, 3, 4, 5]
print(f"加倍后的数字: {[double(x) for x in numbers]}")

PEP 709: 推导式内联

推导式性能优化
Python 3.12通过PEP 709对推导式（列表、字典、集合推导式）进行了优化，使其在内部实现上更高效，减少了函数调用的开销。
示例代码：推导式性能优化
import time

# 示例1: 列表推导式性能测试
print("列表推导式性能测试:")
start_time = time.time()
# 大型列表推导式
squares = [x**2 for x in range(1000000)]
list_time = time.time() - start_time
print(f"生成100万个平方数耗时: {list_time:.4f}秒")

# 示例2: 字典推导式
print("\n字典推导式示例:")
words = ["apple", "banana", "cherry", "date", "elderberry"]
word_lengths = {word: len(word) for word in words}
print(f"单词长度字典: {word_lengths}")

# 示例3: 嵌套推导式
print("\n嵌套推导式示例:")
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flattened = [num for row in matrix for num in row]
print(f"扁平化矩阵: {flattened}")

# 示例4: 带条件的推导式
print("\n带条件的推导式示例:")
even_squares = [x**2 for x in range(20) if x % 2 == 0]
print(f"前20个偶数平方数: {even_squares}")
