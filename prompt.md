## 核心需求

请用 **HTML + CSS + JavaScript（原生，无任何第三方框架/库依赖）** 开发一个纯前端迷宫最短路径可视化应用，完整实现以下功能，代码结构清晰、注释完整、界面美观、交互友好，适配现代浏览器：

---

## 一、功能模块要求

### 1. 迷宫生成与编辑

- 支持**随机生成迷宫**（可配置迷宫大小：行/列，默认15×15，最大20×20），保证起点(0,0)、终点(n-1,m-1)可通行，且至少存在一条通路
- 支持**手动编辑迷宫**：点击格子切换「通路(0)」/「墙壁(1)」，可重置迷宫、清空迷宫
- 支持**自定义起点/终点**：可拖拽或点击设置起点（绿色）、终点（红色），默认左上角为起点、右下角为终点

### 2. 算法实现（核心要求）

必须完整实现以下 **6种搜索算法**，每种算法独立封装，逻辑严谨，符合AI搜索算法标准定义：

1.  **DFS（深度优先搜索）**：可视化搜索树，标注已访问节点、回溯路径，最终输出最短路径
2.  **BFS（广度优先搜索）**：可视化搜索树，分层扩展，保证最短路径
3.  **Dijkstra（迪杰斯特拉算法）**：可视化搜索过程，标注每个节点的当前最短距离g(n)，支持自定义边权（默认每步权值1，可设置不同权值迷宫）
4.  **A\*（A星算法）**：可视化搜索过程，标注每个节点的g(n)（已走代价）、h(n)（曼哈顿距离启发值）、f(n)=g(n)+h(n)，保证最短路径
5.  **Weighted A\*（加权A星算法）**：作为A*的对比，支持自定义权重W（1<W<∞，默认W=1.5），可视化f(n)=g(n)+W×h(n)，对比与标准A*的搜索效率、路径差异
6.  **LRTA\*（实时学习A星，在线搜索算法）**：作为在线搜索扩展，模拟**局部可观测环境**（仅能看到当前节点周围4格，初始未知全局地图），可视化探索过程、H(n)代价表更新、回头路/绕路过程，最终收敛到最短路径

### 3. 可视化要求（核心交互）

- **统一可视化面板**：以网格形式渲染迷宫，不同状态用不同颜色区分：
  - 墙壁：深灰色
  - 通路：白色
  - 起点：绿色
  - 终点：红色
  - 已访问/搜索中节点：浅蓝色（DFS/BFS标注搜索树层级，Dijkstra/A\*标注g/h/f值）
  - 回溯节点：浅橙色（DFS/LRTA\*）
  - 最终最短路径：黄色（高亮，带箭头标注方向）
  - LRTA\*当前位置：蓝色高亮，标注当前H(n)值
- **DFS/BFS专属**：可视化搜索树结构，可展开/折叠节点，标注父节点关系，直观展示搜索顺序
- **Dijkstra/A*/Weighted A*专属**：每个搜索节点悬浮显示g(n)、h(n)、f(n)数值，实时更新启发式函数情况，对比不同算法的搜索范围差异
- **LRTA\*专属**：可视化「已探索区域」（浅灰色）、「未知区域」（深灰色），模拟盲人探路过程，标注H(n)更新日志
- **动画控制**：支持**播放/暂停/重置**，可调节动画速度（慢/中/快三档，默认中速），支持单步执行，方便观察每一步搜索过程

### 4. 对比分析功能

- 支持**多算法并行对比**：可同时选择2-4种算法，在同一迷宫中同步执行，对比搜索时间、搜索节点数、路径长度、搜索范围
- 生成**算法对比报告**：以表格形式展示每种算法的「搜索节点数」「路径长度」「执行时间」「是否最优」，直观对比差异
- 支持**启发式函数对比**：A*与Weighted A*的h(n)、f(n)实时对比，展示权重对搜索效率的影响

### 5. 交互与界面

- 顶部导航栏：算法选择区（6种算法复选框）、迷宫操作区（生成/编辑/重置）、动画控制区（播放/暂停/速度调节）
- 左侧控制面板：迷宫大小设置、权值设置、A*权重W设置、LRTA*模式开关（在线/离线切换）
- 右侧信息面板：实时显示当前算法、搜索进度、节点数值、日志信息
- 底部对比面板：多算法对比表格、可视化搜索范围热力图
- 要求界面美观，可以尽可能多使用毛玻璃效果

1.  **纯原生实现**：仅使用HTML、CSS、JavaScript，不依赖React/Vue/Angular等框架，不引入jQuery、Three.js等第三方库
2.  **性能优化**：迷宫最大20×20时，算法执行流畅，动画无卡顿，避免内存泄漏
3.  **样式规范**：使用Tailwind CSS风格的类名（或原生CSS，结构清晰），界面美观、现代，符合前端审美
4.  **兼容性**：兼容Chrome、Firefox、Edge等主流浏览器，无兼容性问题
5.  分离css、html、js文件以保持代码结构清晰
6.  将所有生成的文件放在V0.5文件夹中

### 补充说明

- 迷宫坐标：使用0-based索引，起点(0,0)对应题目中的(1,1)，终点(n-1,m-1)对应题目中的(n,m)
- 启发式函数：A\*默认使用**曼哈顿距离**（适配上下左右移动），可切换欧几里得距离（可选）
- LRTA\*实现：严格遵循在线搜索逻辑，初始仅可见起点周围，每走一步更新H(n)，模拟真实探索过程
- **最短路径验证**：所有离线算法（DFS/BFS/Dijkstra/A*/Weighted A*）保证输出最短路径，LRTA\*最终收敛到最短路径
